Explanation:
The best solution in this scenario is option (C). AWS Systems Manager Patch Manager provides an easy way to deploy OS patches across multiple EC2 instances at once, reducing the effort required to mitigate the security risk. Additionally, setting up AWS Config to record and track any configuration changes ensures that all changes are properly documented for audit purposes. Option (A) is not suitable because it involves using Amazon QuickSight with Kibana, which may not be necessary in this scenario. Option (B) requires setting up AWS Systems Manager Patch Manager, which can be time-consuming and error-prone. Option (D) does not provide a centralized solution for managing patches across multiple EC2 instances.
